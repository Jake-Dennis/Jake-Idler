<objective>
Extract all inline CSS from apps/server/src/routes/web.ts into separate .css files served via Express static middleware.

**Purpose:** Unblock all subsequent phases (animation pipeline, screen effects, particle system) by separating CSS from JS. Enables future phases to edit animation keyframes and UI styles in standalone files without touching the monolithic web.ts template literals.

**Output:**
- `apps/server/static/css/login.css` — login/hero-select page styles
- `apps/server/static/css/game.css` — game page styles (arena, combat, equipment, party)
- `apps/server/src/index.ts` — +1 line for express.static middleware
- `apps/server/src/routes/web.ts` — inline `<style>` blocks replaced with `<link>` tags

**Constraints (per D-01 through D-04):**
- D-01, D-02: Extract game CSS → game.css, login CSS → login.css (separate files per purpose)
- D-03: Reference via `<link>` tags only — NO behavioral changes to CSS values/selectors
- D-04: Same CSS, same selectors, same values — exact copy, no reformatting or refactoring
- D-05: HTML templates stay inline — only CSS extraction this phase
- D-06: JS stays inline — extraction deferred
- D-07: Font and Lucide CDN links remain in `<head>` — not touched
- D-08: Static directory served via `express.static` at `/static` path
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Express static | The `/static` path serves files from disk to the browser with no authentication |
| Web.ts template → Client | HTML templates are generated server-side and sent to the browser |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `/static` express.static middleware | accept | CSS files contain zero executable logic. `express.static` has no server-side execution; path traversal is mitigated by Express's built-in `serve-static` which normalizes paths. CSS-only content has no XSS vector via `<style>` or `<link>` in modern browsers. Static file serving for non-executable content is an accepted risk. |
| T-01-02 | Information Disclosure | `/static/css/game.css` | accept | CSS contains no secrets, credentials, or PII. Selectors/class names are already visible in the browser via DevTools from the previous inline style — this is not a new disclosure. |
</threat_model>
