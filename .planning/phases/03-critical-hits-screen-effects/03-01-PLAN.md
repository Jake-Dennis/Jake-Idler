## Plan 03-01: Critical Hits & Screen Effects

### Task 1: Add screen shake CSS

Add this to `apps/server/static/css/game.css`:

`.animate-shake-screen{animation:screenShake .3s ease} .animate-shake-screen.crit{animation-duration:.5s} @keyframes screenShake{0%{transform:translate3d(0,0,0)}10%{transform:translate3d(-4px,-2px,0)}20%{transform:translate3d(4px,2px,0)}30%{transform:translate3d(-3px,1px,0)}40%{transform:translate3d(3px,-1px,0)}50%{transform:translate3d(-2px,0,0)}60%{transform:translate3d(2px,0,0)}100%{transform:translate3d(0,0,0)}}`

### Task 2: Add hit-stop effect

Add to game.css: `.animate-hit-stop{animation:hitStop .08s ease} @keyframes hitStop{0%,100%{animation-play-state:running}50%{animation-play-state:paused}}`

### Task 3: Add vignette flash CSS

Add to game.css: `.vignette-flash{position:absolute;inset:0;pointer-events:none;z-index:10;border-radius:6px;opacity:0;transition:opacity .6s ease} .vignette-flash.show{opacity:1} .vignette-flash.crit{background:radial-gradient(ellipse at center,transparent 50%,rgba(139,26,26,.4) 100%)} .vignette-flash.round{background:radial-gradient(ellipse at center,transparent 50%,rgba(0,0,0,.6) 100%)} .vignette-flash.flash{animation:vignetteFlash .2s ease} @keyframes vignetteFlash{0%{opacity:0}50%{opacity:1}100%{opacity:0}}`

### Task 4: Wire effects in web.ts

In `animateTransition` (or the new AnimationQueue integration), add after each damage application:
- If crit: add `.animate-shake-screen .crit` to the arena, add `.vignette-flash.crit.show`, add enhanced float text with gold glow (1.5x, text-shadow)
- If normal heavy hit: add `.animate-shake-screen` to arena
- On round transition: flash vignette
- On melee impact: brief hit-stop class on attacking entity

Use setTimeout(200ms) to remove shake/vignette classes.
